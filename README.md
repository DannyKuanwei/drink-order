# 🧋 員工訂飲料

單頁網頁 + Supabase 即時同步的團體訂飲料工具。員工各自手機開同一網址，即時看到同一張彙總單。

## 線上網址
- **使用（員工訂）**：https://dannykuanwei.github.io/drink-order/
- 程式：單一 `index.html`，放在 GitHub Pages，後端用 Supabase（`drink_orders` 表）。

## 怎麼用
1. 填名字（會記住）
2. 選店家 → 選品項 → 容量／甜度／冰塊 → 加料備註（選填）
3. 送出。所有人即時看到同一張單、各品項統計、總杯數與金額
4. 每間店有「📋 複製給店家」：自動把該店訂單按品項+甜冰加總成可貼文字
5. 訂錯按該筆右邊的 × 刪除
6. 每天自動分批（依日期），今天明天不混

## 🎲 骰子規則（幫拿不定主意的人決定）
- **每人限骰 2 次**，依「**名字**」各自計算（要先填名字才能骰）
- 按一下 → 隨機決定整杯：**品項＋容量＋甜度＋冰塊**
- 次數存在瀏覽器、依當天批次計算，**重新整理頁面不會重置**（不能靠刷新偷骰）
- **可以「取消重選」**：骰到不喜歡的可按「↩ 取消重選」清空、改自己選
- ⚠️ **取消不退還骰子次數**：骰兩次就是兩次，取消只是讓你不被骰到的結果綁住
- 規則一句話：**一人骰兩次，骰完不能再骰，但可以取消**

### 改菜單 / 價格 / 電話
編輯 `index.html` 裡的 `SHOPS` 陣列：
```js
{ id:'店代號', name:'店名', phone:'07-xxxxxxx', menu:[
  {name:'品項', m:中杯價, l:大杯價}, ...
]}
```
改完 commit + push，GitHub Pages 約 1 分鐘後自動更新。
